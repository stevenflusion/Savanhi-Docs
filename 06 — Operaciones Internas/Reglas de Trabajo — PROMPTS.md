# Reglas de Trabajo — PROMPTS

> Estos son los puntos no discutibles del equipo Savanhi. Todo el equipo debe leerlos y respetarlos.
> 

---

## Puntos NO discutibles

<aside>
🔴

**INCONSISTENCIAS**: Si hay inconsistencias en cualquier archivo o documento, **no se deduce NUNCA**.

Lo que se hace es: **PARAR y preguntar** antes de continuar.

</aside>

<aside>
📊

**INVESTIGACIÓN PRIMERO**: La investigación **siempre** va antes que la escritura o la codificación.

Nunca se escribe ni se programa sin antes investigar lo necesario.

</aside>

<aside>
🔧

**MANEJO DE ERRORES**: Cuando haya un error, se intenta resolver. Pero si la solución no funciona:

1. **PARAR**
2. **ANALIZAR** el error
3. **INVESTIGAR** causas
4. **ANALIZAR** la o las posibles soluciones
5. **ARREGLARLO**

No se salta ningún paso.

</aside>

<aside>
🙋

**PEDIR AYUDA**: Si se necesita algo del equipo (información, decisión, recurso), **PARAR y pedirlo**.

No se siente pena. Es más costoso asumir que preguntar.

</aside>

<aside>
✅

**CIERRE DE TAREAS**: Una tarea **NO** puede darse por finalizada hasta no hacer todas las pruebas posibles.

Sin pruebas = sin cierre.

</aside>

---

## Resumen rápido

| Regla | Acción |
| --- | --- |
| Hay inconsistencia | PARAR → preguntar |
| Voy a escribir o codificar | INVESTIGAR primero |
| Encontré un error y mi solución no funcionó | PARAR → analizar → investigar → resolver |
| Necesito algo del equipo | PARAR → pedir sin pena |
| Terminé una tarea | Hacer TODAS las pruebas posibles primero |
